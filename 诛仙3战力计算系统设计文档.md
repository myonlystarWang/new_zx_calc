# 诛仙3战力计算系统设计文档

## 1. 引言

本文档旨在详细描述《诛仙3》游戏角色战力计算系统的设计方案，包括数据模型、核心计算逻辑以及战力评估方法。

系统将围绕以下核心实体进行数据建模：

### 2.2 技能 (Skill)
这里有一个问题：每一个职业对应了三个阵营（仙、佛、魔），技能也需要区分阵营。那么我们这个数据结构应该如何设计？

**解决方案：**

为了清晰地表示职业、阵营和技能之间的关系，并方便数据管理和查询，我建议对技能的数据结构进行调整。我们可以在 `skills.json` 文件中，将原先直接以 `ClassID` 作为键的结构，修改为以 `ClassID_FactionID` 的组合形式作为键，或者在每个职业的技能列表中，再按阵营进行一次嵌套。

考虑到查询效率和数据直观性，我更倾向于**在每个职业的技能列表中，再按阵营进行一次嵌套**。这样，`skills.json` 的顶层键仍然是 `ClassID`，其对应的值是一个对象，该对象的键是阵营ID（例如 `XIAN`, `FO`, `MO`），值是对应阵营的技能列表。

**`skills.json` 调整后的结构示例：**

```json
{
  "QING_YUN": { // 职业ID，例如青云
    "XIAN": [ // 仙阵营技能列表
      {
        "SkillID": "QY_XIAN_SKILL_001",
        "SkillName": "仙·神剑御雷真诀",
        // ...其他技能属性，如 SkillImportanceWeight, SkillFrequency, Cooldown, SkillBonusAttributes 等
        "Faction": "XIAN" // 可以显式指明阵营，便于校验
      }
      // ...更多仙青云技能

    "FO": [ // 佛阵营技能列表
      {
        "SkillID": "QY_FO_SKILL_001",
        "SkillName": "佛·大道无形",
        // ...其他技能属性
        "Faction": "FO"
      }
      // ...更多佛青云技能
    ],
    "MO": [ // 魔阵营技能列表
      {
        "SkillID": "QY_MO_SKILL_001",
        "SkillName": "魔·天外飞仙",
        // ...其他技能属性
        "Faction": "MO"
      }
      // ...更多魔青云技能
    ]
  },
  "GUI_WANG": { // 职业ID，例如鬼王
    "XIAN": [
      // ...仙鬼王技能
    ],
    "FO": [
      // ...佛鬼王技能
    ],
    "MO": [
      // ...魔鬼王技能

```

**优点：**

1.  **结构清晰**：职业、阵营、技能的层级关系一目了然。
2.  **查询方便**：可以先根据职业ID定位到职业，再根据阵营ID获取特定阵营的技能列表。
3.  **扩展性好**：如果未来出现新的阵营或职业，可以方便地添加。
4.  **数据一致性**：`SkillID` 可以设计为包含职业和阵营信息（如 `QY_XIAN_SKILL_001`），进一步增强唯一性和可读性。

## 3. 伤害计算公式

此处将详细定义角色使用特定技能攻击特定怪物时的伤害计算逻辑。

**基础伤害计算（示例）：**
`最小基础伤害 = (角色最小攻击 * (1 + 技能附加攻击百分比 / 100)) + 技能附加固定攻击 + (角色气血 * 技能附加气血百分比 / 100) + (角色真气 * 技能附加真气百分比 / 100) + (角色防御 * 技能附加防御百分比 / 100) `
`最大基础伤害 = (角色最大攻击 * (1 + 技能附加攻击百分比 / 100)) + 技能附加固定攻击 + (角色气血 * 技能附加气血百分比 / 100) + (角色真气 * 技能附加真气百分比 / 100) + (角色防御 * 技能附加防御百分比 / 100) `

**最终伤害：**
`最小最终伤害 = 最小基础伤害 * ((角色暴击伤害 + 怪物被暴击伤害增益 + 技能附加暴击伤害- 怪物减爆伤) / 100) * (1 + 技能伤害增加倍数 / 100) * (1 + 角色对怪增伤 / 100) * (1 + 怪物受到伤害增益 / 100) * (1 + 专注增益 / 100) * (1 + 巫咒增益 / 100)`
`最大最终伤害 = 最大基础伤害 * ((角色暴击伤害 + 怪物被暴击伤害增益 + 技能附加暴击伤害- 怪物减爆伤) / 100) * (1 + 技能伤害增加倍数 / 100) * (1 + 角色对怪增伤 / 100) * (1 + 怪物受到伤害增益 / 100) * (1 + 专注增益 / 100) * (1 + 巫咒增益 / 100)`

## 4. 战力评估方法

### 4.1 单个副本战力

1.  **遍历副本内所有关卡的所有怪物/Boss。**
2.  **针对每个怪物/Boss，遍历角色所有可用技能。**
3.  **使用第3节的伤害计算公式，计算角色使用每个技能对当前怪物/Boss造成的最小和最大伤害。**
    -   此时需要考虑固定的战斗增益，或用户填写的战斗增益。
4.  **伤害汇总与评估：**
    -   **方法一（基于平均伤害）：** 计算每个技能对每个怪物的平均伤害（(最小伤害 + 最大伤害) / 2）。然后可以根据技能的重要程度（例如：主力输出技能权重更高）或使用频率进行加权平均，得到一个综合伤害值。
    -   **方法二（基于击杀效率）：** 估算击杀每个怪物/Boss所需的时间或技能次数。例如，`击杀时间 ≈ 怪物气血 / 平均每秒伤害 (DPS)`。副本的总战力可以与通关该副本的预估总时间或总消耗资源负相关。------------暂时无法计算
    -   **方法三（基于特定指标）：** 选取一些关键指标，如对Boss的最高爆发伤害、持续输出能力等，综合评估。
        -   **具体评估方案建议：**
            1.  **Boss最高爆发伤害：** 计算角色所有技能中，单次对Boss造成的最高平均伤害。此指标反映角色的瞬间输出上限。
            2.  **Boss战持续输出能力 (模拟)：**
                *   设定一个标准的Boss战时长（例如60秒）。
                *   根据各技能的冷却时间（需补充数据）、施法时间（可简化或预设）、以及技能重要程度/使用频率，模拟在这段时间内各技能的使用次数。
                *   `Boss战预估总伤害 = Σ (技能平均伤害 * 模拟使用次数)`
                *   `Boss战DPS = Boss战预估总伤害 / 标准Boss战时长`
            3.  **小怪清理效率 (模拟)：**
                *   选取副本中代表性的小怪。
                *   计算使用AOE技能（范围攻击技能，需补充数据）或高频率单体技能击杀该小怪的平均时间。
                *   `小怪清理效率评分 = 1 / 平均击杀时间` (数值越大越好)
            4.  **综合指标评分：** `综合指标评分 = (Boss最高爆发伤害 * 权重A) + (Boss战DPS * 权重B) + (小怪清理效率评分 * 权重C)`
                *   权重A, B, C 需要根据游戏实际体验和不同职业特性进行调整，例如爆发型职业的权重A可以更高。
5.  **副本综合战力评分：** 将上述计算得到的针对副本内所有怪物/Boss的伤害数据或效率数据，通过一个预设的算法（可能需要根据游戏经验调整权重）聚合成一个单一的数值，作为角色对该副本的综合战力评分。
    -   **详细算法设计方案建议：**
        1.  **怪物/Boss重要性权重：** 为副本中的每个怪物/Boss设定一个重要性权重 (MonsterWeight)。通常Boss的权重远高于小怪。
        2.  **单个怪物/Boss的威胁度/挑战度评分：**
            *   基于方法一（平均伤害）：`怪物挑战度 = 1 / (Σ (技能平均伤害对该怪物 * 技能重要程度权重) / Σ 技能重要程度权重)` (即综合平均伤害的倒数，伤害越高，挑战度越低，战力贡献越高)
            *   或者，如果采用方法三的指标，可以针对每个Boss计算其对应的指标得分。
        3.  **副本内所有怪物/Boss的加权战力贡献：**
            *   `副本综合战力评分 = Σ (单个怪物/Boss的挑战度评分 * MonsterWeight)` (如果挑战度评分是伤害的倒数)
            *   或者 `副本综合战力评分 = Σ (针对该Boss的方法三综合指标评分 * MonsterWeight)`
        4.  **标准化（可选）：** 为了方便比较不同副本的战力，可以将评分进行标准化处理，例如映射到0-10000的范围。
        5.  **核心思路：** 角色对副本中越重要的怪物（高权重Boss）能造成越高的有效伤害（或表现出更高的击杀效率/关键指标），则其在该副本的战力评分越高。
    
### 4.2 所有副本综合战力

1.  **计算角色对每个独立副本的综合战力（如4.1所述）。**
2.  **加权平均：** 根据不同副本的重要性或难度，为每个副本的战力评分赋予不同的权重。
    `所有副本综合战力 = Σ (单个副本战力评分 * 副本权重) / Σ (副本权重)`
    -   例如，日常、高奖励的副本权重可以设置得更高。


### 4.3 最新战力计算方法（全）    
1. 怪物战力计算
公式：  
怪物战力=∑(技能平均伤害×技能权重)

解释：
怪物的战力是它所有技能的加权伤害之和。  
技能平均伤害：每个技能造成的平均伤害值（你提到已知）。  

技能权重：反映技能在战斗中的重要性，权重越高，该技能对怪物战力的贡献越大。

示例：
假设一个怪物有三个技能：  
技能A：平均伤害100，权重1.0 → 贡献100  

技能B：平均伤害80，权重0.5 → 贡献40  

技能C：平均伤害120，权重1.2 → 贡献144  

怪物战力 = 100 + 40 + 144 = 284

2. 副本战力评分计算
公式：  
副本战力评分=平均(所有怪物战力)

解释：
副本战力评分是该副本内所有怪物的战力的平均值。  
使用平均值可以平衡副本内怪物数量的差异，避免因怪物数量多少而影响评分。

示例：
假设一个副本有三个怪物，战力分别为140、216、250：  
副本基础战力 = (140 + 216 + 250) / 3 ≈ 202

3. 总战力评分计算
公式：  
总战力评分=加权平均(所有副本战力评分)

解释：
总战力评分是对所有副本战力评分的加权平均值，每一个副本都有一个权重参数  
它可以用来评估整个游戏的难度水平或玩家的整体实力。



## 5. 用户交互流程

1.  **用户填写/更新角色属性，选择职业。**
2.  **系统加载角色职业对应的技能列表。**
3.  **系统加载计算所有副本。**
4.  **用户填写/选择战斗增益（可选，可使用默认值）。**
5.  **系统执行计算。**
6.  **展示计算结果：**
    -   各技能对各怪物的详细最大最小最终伤害范围。
    -   单个副本的综合战力评分。
    -   （如果选择）所有副本的综合战力评分。
7.  数据输入便利性 ：如何让用户方便、准确地输入大量的游戏数据（角色属性、技能等级、装备细节等）会是影响用户体验的关键。可以考虑：
    - 提供清晰的输入引导和示例。
    - 允许用户保存和加载多套角色配置。
    - （更高级）研究是否有办法从游戏客户端（通过插件、截图OCR等方式，但这通常较复杂且可能违反游戏TOS）半自动获取数据。

## 7. 技术风格建议
目前流行的**“游戏暗黑风” + “毛玻璃特效” (Glassmorphism)**
1. 视觉升级：采用深色背景搭配霓虹渐变色，符合游戏玩家审美。
2. 交互体验：输入数值时实时计算，配有滑块和数字输入，操作更顺滑。
3. 数据可视化：增加了角色属性“多维图”（雷达图）和评级徽章系统（SSS/SS/S/A/B/C），让战力结果更直观。
4. 移动端适配：完美支持手机和电脑端访问。

配色方案：使用了 Slate-900 作为深色背景，搭配 Cyan (青色) 和 Purple (紫色) 作为强调色。这种“赛博朋克/电竞”风格非常适合游戏战力计算器。

输入体验：
将每个属性做成了独立的卡片。同时提供了数字输入框和滑动条。用户既可以精准输入（如直接输入 12345），也可以快速滑动查看战力变化趋势。

结果反馈：
右侧（或移动端的下方）固定展示总战力。
根据数值自动计算评级徽章（SSS, SS, S...），给用户正向反馈。

增加了雷达图，让数据不再只是枯燥的数字。
响应式布局：使用了 CSS Grid，在电脑上是左输入右结果，在手机上自动变为单列布局，确保手机用户体验良好。
