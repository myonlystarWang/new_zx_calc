# 技能附加信息悬浮显示与UI优化设计方案

**日期**：2026-03-14
**版本号**：V1.0

## 1. 需求分析
现有战力计算器界面在“交互”与“信息呈现”上已具备较高的完成度（暗夜蓝配色、玻璃拟物感）。但仍存在一定的优化空间：
1. **技能数据展示太占空间**：每个副本下方展示技能伤害时，由于 `skills.json` 中配置了众多属性（段数、施法时间、冷却、附加攻击等），如果全部平铺会致使表格异常冗长。
2. **需要悬浮提示**：用户要求当选定职业和阵营后，必须有一种不占用主干空间的方式查看技能的附加信息。

## 2. 交互与UI优化建议
基于前端 Vibe Coding 理念：
- **增加气泡悬浮窗（Tooltip）**：在技能名称旁边增加一个小的 `Info` 图标，利用 CSS 的悬停伪类触发绝对定位的浮层，展示该技能所有附加属性。
- **强化视觉深度与反馈**：
  - 提升按钮与卡片在 `hover` 时的过渡流畅度。
  - 针对“综合战力评分”等重点数据区增加微妙的光晕闪烁动画。

## 3. 技术实现方案
- **相关文件**：`web_app/src/components/business/DungeonDetail.tsx`。
- **悬停能力与定位优化**：原先简单的 `group-hover` 在复杂的表格和具有 `overflow-hidden` 属性的父容器中容易引发滚动条和位置错乱。优化方案改为：使用 React 组件级别的状态（如 `onMouseEnter` / `onMouseLeave` / `onMouseMove`）捕捉鼠标位置，并在离最外层更近的层级（或是 fixed 定位）渲染单独的 Tooltip 组件，使其精准跟随鼠标或定位在图标附近，不受深层外层 `overflow` 限制。
- **数据字段展示要求**：
  - 标题统一为：“技能详细信息”
  - 展示内容包含：附加攻击比 (AttackRatio)、附加固定攻击 (Bonus)、附加气血比 (HP_Ratio)、附加真气比 (MP_Ratio)、附加爆伤 (CritDmgRatio)；伤害增加倍数 (SkillDamageBonus)；重要性 (依据 SkillImportanceWeight 判断)；使用频次 (依据 CastFrequency 判断)。

## 4. 最终呈现流程
1. 鼠标悬停到副本卡片下方对应技能的 `Info` 图标上。
2. 弹出毛玻璃样式的暗色卡片，卡片位置智能计算并紧随鼠标/图标附近，不会被父级容器裁切或触发整体滚动条。
3. 卡片内采用列表形式展现其扩展加成数据，内容精简且符合上述配置。
