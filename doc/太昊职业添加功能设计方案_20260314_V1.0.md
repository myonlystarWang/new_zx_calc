# 太昊职业添加功能设计方案

## 1. 背景
应用户需求，在“诛仙3战力计算系统”中添加新职业“太昊”，并配置其核心技能“天地绝”的三个版本（仙、佛、魔）。

## 2. 需求描述
- **职业**：神族 - 太昊
- **技能**：天地绝
- **阵营对应后缀**：
  - 仙：·玄
  - 佛：·禅
  - 魔：·煞
- **技能属性**：
  - 附加攻击比：200%
  - 附加固定攻击：2750
  - 附加气血比：26%
  - 附加真气比：26%
  - 冷却时间：0s
  - 施放时间：0.3s
  - 技能频率：0.9
  - 技能权重：0.9

## 3. 设计方案

### 3.1 数据层变更
- **classes.json**: 确认“太昊”职业已存在，Race为“神族”。
- **skills.json**: 增加 `TAI_HAO` 节点，配置 `XIAN`、`FO`、`MO` 阵营下的技能数据。

### 3.2 SkillID 命名规则
为了便于后续扩展和维护，技能 ID (SkillID) 需遵循以下命名规范：

1.  **普通技能**：`职业拼音缩写_阵营_SKILL_技能拼音缩写`
    *   示例：`TH_XIAN_SKILL_TDJX` (太昊·仙·天地绝)
2.  **四代技能 (赤乌、玄烛)**：`职业拼音缩写_阵营_SKILL_技能拼音缩写_品质缩写`
    *   **品质缩写说明**：
        *   曦日：`XR`
        *   皓月：`HY`
        *   荧炬：`YJ`
    *   示例：`NY_XIAN_SKILL_DYFT_XR` (涅羽·仙·大业浮屠·曦日)

### 3.3 技能数据结构 (skills.json)
```json
"TAI_HAO": {
    "XIAN": [{
        "SkillID": "TH_XIAN_SKILL_TDJX",
        "SkillName": "天地绝·玄",
        "RequiredClass": "TAI_HAO",
        "Faction": "XIAN",
        "SkillBonusAttributes": {
            "SkillAttackPercentBonus": 200,
            "SkillAttackFixedBonus": 2750,
            "SkillHealthPercentBonus": 26,
            "SkillManaPercentBonus": 26,
            "SkillCriticalDamagePercentBonus": 100,
            "SkillDamageBonus": 1
        },
        ...
    }],
    ...
}
```

### 3.3 计算逻辑验证
- `calculator.ts` 已支持 `SkillHealthPercentBonus` 和 `SkillManaPercentBonus`，无需修改逻辑。

## 4. 影响评估
- 前端 `ClassSelector` 将自动显示“太昊”职业（因为 `skills.json` 中有了数据）。
- `FactionSelector` 将允许为太昊选择阵营并计算对应技能伤害。
