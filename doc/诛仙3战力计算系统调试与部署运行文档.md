# 诛仙3战力计算系统调试与部署运行文档

## 1. 项目简介
本项目是一个基于 **Vite + React + TailwindCSS** 的前端 SPA，并通过 **Cloudflare Pages Functions** 提供轻量级的卡密验证与数据脱敏能力，用于《诛仙3》游戏的战力模拟与伤害计算。

## 2. 环境要求
- **Node.js**: 建议版本 v18.0.0 或更高
- **npm**: 建议版本 v9.0.0 或更高

# 3. 本地开发与调试
进入 `web_app` 目录进行操作：

### 3.1 仅前端界面调试 (Vite)
如果您只修改 UI 样式或简单的计算公式，不涉及权限数据逻辑：
```bash
npm run dev
```
访问地址：`http://localhost:5173`。

### 3.2 全链路功能调试 (Wrangler)
如果您修改了卡密校验、数据脱敏逻辑或需要测试高级版数据：
```bash
# 1. 编译前端
npm run build

# 2. 模拟本地 API 环境
# 本地测试建议加 --local，避免联网等待或鉴权卡住
# 本地测试时可以使用 --binding VIP_CODES="test1,test2" 来手动指定白名单
npx wrangler pages dev ./dist --local --compatibility-flags=nodejs_compat --binding VIP_CODES="test1"
```
访问地址：`http://localhost:8788`。

也可以直接使用脚本：
```bash
npm run dev:full
```

## 4. 数据目录说明
**重要变更**：为防止数据滥用，核心 JSON 文件已从 `public/` 迁移至 `functions/data/`，并在 Functions 中通过 `import` 静态加载。

| 文件名 | 描述 |
| :--- | :--- |
| `functions/data/classes.json` | 职业基本信息与技能关联 |
| `functions/data/skills.json` | 详细技能伤害参数与权重 |
| `functions/data/dungeons.json` | 副本列表与元数据 |
| `functions/data/dungeons_monsters.json` | 副本关联的怪物与Boss属性 |
| `functions/data/combat_buffs.json` | 战斗增益（药、技能Buff等） |
| `functions/data/rank_config.json` | 战力评级配置 |

`public/game_data/` 目录目前仅保留占位文件（`.gitkeep`），不再存放有效数据。

## 5. 部署方式
由于引入了 **Pages Functions**，请使用 **Wrangler 部署**（推荐）或 **Git 集成部署**。拖拽上传的“直接上传模式”不支持 Functions。

### 5.1 第一步：本地打包构建
1. 进入 `web_app` 目录。
2. 执行构建：
   ```bash
   npm run build
   ```

### 5.2 第二步：Wrangler 部署（推荐）
```bash
npx wrangler pages deploy dist --project-name zhuxiancalculator
```
Wrangler 会自动读取项目根目录下的 `functions/` 并一同部署。

也可以直接使用脚本：
```bash
npm run deploy
```

### 5.3 Git 集成部署（可选）
在 Cloudflare Pages 中绑定 Git 仓库后，构建命令填写：
```bash
npm run build
```
输出目录填写：
```
dist
```
Functions 会从仓库根目录的 `functions/` 自动识别部署。

### 5.4 环境变量配置
在项目设置页：**Settings** -> **Environment variables** 添加 `VIP_CODES`（英文逗号分隔）。

### 5.5 如何更新网站？
以后如果您修改了游戏数据（JSON 文件）或代码：
1. 执行 `npm run build` 重新生成 `dist`。
2. 使用 `wrangler pages deploy dist --project-name <你的项目名>` 重新部署，或提交代码触发 Git 集成部署。

### 5.6 常见问题排查
- **本地函数不生效**：请用 `wrangler pages dev` 启动本地环境，而不是单独的 `vite dev`。
- **构建报错**：确保在 `npm run build` 之前执行过 `npm install`。
- **数据没变**：请确认您在 `npm run build` 之前已经保存了对 JSON 文件的修改。
