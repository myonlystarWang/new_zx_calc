# 诛仙3战力计算系统调试与部署运行文档

## 1. 项目简介
本项目是一个基于 **Vite + React + TailwindCSS** 开发的纯前端单页面应用（SPA），专门用于《诛仙3》游戏的战力模拟与伤害计算。

## 2. 环境要求
- **Node.js**: 建议版本 v18.0.0 或更高
- **npm**: 建议版本 v9.0.0 或更高

## 3. 本地开发与调试
进入 `web_app` 目录进行操作：

```bash
# 1. 进入前端代码目录
cd web_app

# 2. 安装依赖（首次运行或更新 package.json 后）
npm install

# 3. 启动本地开发服务器
npm run dev
```
启动后，默认访问地址为：`http://localhost:5173`。

## 4. 数据目录说明
项目的数据文件存放在 `web_app/public/game_data` 中。

| 文件名 | 描述 |
| :--- | :--- |
| `classes.json` | 职业基本信息与技能关联 |
| `skills.json` | 详细技能伤害参数与权重 |
| `dungeons.json` | 副本列表与元数据 |
| `dungeons_monsters.json` | 副本关联的怪物与Boss属性 |
| `combat_buffs.json` | 战斗增益（药、技能Buff等） |
| `rank_config.json` | 战力评级（SSS/SS/S等）阈值配置 |

## 5. 部署与日常更新指南 (手动上传模式)

由于本项目是纯前端应用，您在 Cloudflare Pages 上使用的是“直接上传文件夹（Direct Upload）”的方式。这种方式最简单，不需要配置复杂的 Git 仓库同步。

### 5.1 第一步：本地打包构建
在上传之前，必须将源代码编译成浏览器能识别的静态文件。

1. 打开终端，进入前端目录：
   ```bash
   cd d:\王炜\工作\个人工作\wx_proj\new_zx_calc\web_app
   ```
2. 执行构建命令：
   ```bash
   npm run build
   ```
3. **结果确认**：运行完成后，`web_app` 目录下会生成一个名为 **`dist`** 的文件夹。这个文件夹里的内容就是我们要上传到 Cloudflare 的全部。

### 5.2 第二步：手动上传到 Cloudflare Pages
1. 登录 [Cloudflare 控制面板](https://dash.cloudflare.com/)。
2. 点击左侧菜单的 **Workers & Pages** -> **Create application** -> **Pages** -> **Upload assets**。
3. 给项目起个名字（如 `zx3-calc`）。
4. **上传操作**：
   - 将刚才生成的 **`web_app/dist`** 文件夹整个拖入上传框；
   - 或者点击选择文件夹，选中 `dist` 目录。
5. 点击 **Deploy site**。部署完成后，Cloudflare 会提供一个 `pages.dev` 后缀的网址，您可以直接访问。

### 5.3 如何更新网站？
以后如果您修改了游戏数据（JSON文件）或代码：
1. 重复 5.1 的步骤：执行 `npm run build` 重新生成 `dist` 文件夹（这一步很重要，否则上传的是旧内容）。
2. 登录 Cloudflare Pages，找到对应的项目。
3. 点击 **Create a new deployment**。
4. 再次拖入新的 **`dist`** 文件夹即可。

### 5.4 常见问题排查
- **上传后白屏**：确认您上传的是 `dist` 文件夹**里面的内容**，还是带了一个 `dist` 父目录。正确做法是让 `index.html` 位于项目的根路径。
- **构建报错**：确保在 `npm run build` 之前执行过 `npm install`。
- **数据没变**：请确认您在 `npm run build` 之前已经保存了对 JSON 文件的修改。
